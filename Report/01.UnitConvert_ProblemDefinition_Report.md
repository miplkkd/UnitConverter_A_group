> Mom Test 인터뷰 기반 문제 정의 보고서  
> 작성 기준: 페르소나 과거 사실 인터뷰 (3턴)  
> Report 넘버링: `01.`

## 1. Executive Summary

프로그래밍 수업의 부분 단위변환 과제에서, 학생은 **「단위 변환 프로그램을 만든다」**는 표면 정의보다 **「import·환경 막힘 없이 변환을 재현하고, 실패 테스트를 제거하지 않고 통과시킨다」**는 문제를 먼저 해결해야 한다. 본 보고서는 Mom Test 인터뷰 증거를 바탕으로 진짜 문제를 정의하고, 이번 세션 범위와 금지 항목을 명시한다.

**주제 한 문장:** 과제 첫날에 변환 결과를 재현 가능하게 확인하고, import 막힘 없이 테스트가 돌아가게 만드는 것까지가 목표다.

## 3. Mom Test Interview — Past Facts

### Turn 1 — 과제 수령 직후 행동

- 과제 PDF 링크 수령 후 **README부터** 읽음.
- 출력 예시(`meter:2.5` → feet, yard…)와 OCP, SRP, 테스트 요구가 먼저 눈에 들어옴.
- 추가 요구(JSON, 동적 단위 등록)는 「시간 남으면」으로 분류 → **기본만** 하기로 결정.
- `UnitConverter.py` 열어 main()의 if-elif 구조 확인 → 「OCP 안 된다」는 **느낌만**, 쪼개는 방법은 몰랐음.
- 터미널 실행: `meter:2.5` 정상 출력 확인; `abc:1`, `meter:hello` **손으로** 실험 → 입력 검증 필요성 체감.
- **테스트부터 쓰지 않음.** pytest 폴더 구조 헷갈림 → 계산기로 2.5m→feet 손계산만.
- AI는 당일 미사용. 「인터페이스 어떻게 나누지」 검색 → 예제가 길어 포기 → **마감 전날** AI에 예시 질문.

**실제 순서:** README → 코드 열기 → 실행·오입력 실험 → 손계산 확인 → (테스트·리팩터링은 그다음 날)

### Turn 2 — pytest 시도와 포기

- 「테스트 안 하면 감점」 README 문구 → pytest **한 번 시도**.
- `tests/` 폴더 생성, `tests/test_convert.py`에 assert 한 줄 작성.
- 가상환경 없이 `pytest` 실행 → not recognized → `python -m pytest`로 재시도.
- 테스트 수집 후 **ModuleNotFoundError** (UnitConverter import 실패).
- `sys.path`·패키지 구조 검색 → **더 읽기 싫어져 파일 닫음**.

**당시 자기 합리화:**

1. 「pytest는 환경/경로 문제, 변환 로직이 틀리면 테스트도 의미 없음」
2. 「지금 막히는 건 if-elif 리팩터링, assert 한 줄이 아님」
3. 「README 숫자랑 출력만 맞으면 발표 때 덜 당황함」

- conftest, coverage, CI는 **전혀 미착수**.

### Turn 3 — TA 도움과 제출

- 다음 날 아침 TA 부스: **15분 대기**, 2~3분 상담.
- TA 조언: `tests/`와 `UnitConverter.py` 같은 루트, `python -m pytest tests/`, `tests/__init__.py`, main()만 있으면 import해도 테스트할 게 없으니 **함수/클래스 분리**.
- `__init__.py` 추가·명령 변경 후에도 import 에러 → 변환 로직을 `to_meters(value, unit)` 등 **함수로 분리**.
- TA 상담 후 **40분~1시간** 만에 처음 pytest 초록. 테스트 **2개만** 통과 (음수·형식 검증 테스트는 당일 미작성).
- 제출 전날: 음수 테스트 추가 → 코드에 검증 없어 **실패** → **주석 처리 후 제출**.
- import 문제는 함수 분리 + 루트 `python -m pytest`로 해결. `pip install -e .` 미사용.
- JSON·동적 단위 **미제출**. 기본 변환 + 테스트 2개 + 데모 영상으로 **부분 점수**.
- TA 두 번째 질문(검증 테스트 작성법) 못 함 → 친구에게 「이 assert 빼고 제출해도 되냐」만 질문.

## 4. Problem Definition

### 4.1 Surface Problem (금지 — 솔루션 섞인 문장)

> 「OCP·SRP를 만족하는 **단위 변환 프로그램**을 만들고, pytest로 **테스트까지 작성**한다.」

이 정의는 **제품·기능 중심**이며, 학생이 실제로 막히는 지점(import, 재현, 제출 판정)을 반영하지 않는다.

### 4.2 Real Problem (한 문장)

> **main()만 있는 시작 코드에 테스트·설계·제출 마감이 동시에 걸리면, 학생은 첫날 import·환경 막힘에 시간을 쓰고 변환은 손계산으로 대신하며, 마감 직전에는 실패한 테스트를 고치기보다 빼서 제출 판정만 맞추려 한다.**

### 4.3 Mom Test Evidence (인용 3줄)

1. 「`ModuleNotFoundError` … `sys.path`나 패키지 구조 얘기가 나와서 **더 읽기 싶어졌어요**. … **‘일단 변환부터 돌아가게’ 하고 파일 닫았어요**.」
2. 「**‘pytest는 환경/경로 문제인 것 같고, 변환 로직이 틀리면 어차피 테스트도 의미 없음’** … **README 숫자랑 출력이 맞는지만 오늘 확인하면 발표 때 덜 당황함**」
3. 「음수 테스트 실패 → **주석 처리하고 제출** … **‘이거 빼고 제출해도 되냐’**만 물었어요.」

## 8. Session Layers (Rule · Command · Skill · Test Loop)

### 8.1 Rule

- 목표: 변환 재현 + 루트 pytest 실행 + 실패 테스트 주석 처리 금지.
- 범위: meter/feet/yard 변환, 입력 검증(형식·숫자·단위·음수), tests/ 최소 구조.
- main()만 있으면 테스트 불가 → 변환·검증 로직을 import 가능한 함수/클래스로 분리.
- 실행·테스트: 프로젝트 루트에서 `python -m pytest tests/`
- README 비율: 1m = 3.28084ft, 1m = 1.09361yd.

### 8.2 Command (순서 고정)

1. README 비율·검증 요구 확인 → 세션 범위(기본 3단위 + 검증)만 적기
2. `UnitConverter.py` 실행: 정상·오입력·음수 → 현재 동작 기록
3. 변환·검증 로직 import 가능하게 분리 (main은 입출력만)
4. `tests/__init__.py` + `tests/test_convert.py` 생성
5. 루트 `python -m pytest tests/` → import 에러 해소
6. assert: 정상 변환 2~3개 + 오입력 + 음수 → 전부 통과
7. 제출 전: 주석 처리 assert 0개, pytest 초록

### 8.3 Skill (선택) — pytest-import-unblock

- 트리거: ModuleNotFoundError, import 에러
- 절차: cwd 루트 확인 → `python -m pytest tests/` → `tests/__init__.py` → 함수 분리·import 경로 점검
- 금지: pip install -e ., conftest, coverage를 먼저 제안

### 8.4 Test Loop

| Loop | 내용 | Pass | Fail 시 |
|------|------|------|---------|
| A — 실행·재현 | meter/feet/yard 변환 | README 비율·assert 일치 | 코드 수정 → A 재실행 |
| B — import·수집 | `python -m pytest tests/ --collect-only` | 수집됨, import OK | Rule·Command 3~5 → B 재실행 |
| C — 검증·판정 | abc:1, meter:hello, 음수, unknown unit | assert 전부 통과 | 검증 구현 (주석 금지) → C 재실행 |

**Exit:** A + B + C 동시 Pass → 세션 종료.

---

## 9. Conclusion

이번 세션의 완료 정의는 **「완성된 단위 변환 앱」**이 아니라 **「루트에서 pytest 초록 + 검증 테스트 통과」**이다. import·손계산·주석 처리 루프를 끊는 것이 Mom Test가 검증한 1순위 문제다.
