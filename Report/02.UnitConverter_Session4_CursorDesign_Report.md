## 1. Executive Summary

세션 4에서는 Mom Test·PRD(세션 1~2)와 헌법 갱신(세션 3 `.cursorrules`)을 바탕으로 **Cursor 8계층 중 Skill·Command·Hook·Test/Review Loop 검증 기준**을 UnitConverter 도메인에 맞게 구축했다. 구현·`test_d_*` RED 실물은 **미착수** — 다음 세션 권장 작업은 `/tdd-red` → `D-CONV-01` RED + Loop B.

**주제 한 문장:** 단위 변환 과제에서 import·검증 막힘을 끊기 위해, Rule 위에 TDD RED 진입·ECB 리뷰·pytest Loop를 **재현 가능한 아티팩트**로 고정한다.

## 5. hooks.json

```json
sessionStart → .cursor/hooks/session-init.sh
postToolUse (matcher: Write) → .cursor/hooks/hint-after-src-test-write.sh
```

| Hook | 기대 동작 | 현재 |
|------|-----------|------|
| `sessionStart` | UnitConverter 컨텍스트·`/tdd-red` 주입 | 스크립트 **없음** |
| `postToolUse` | `src/`·`tests/` Write 시 pytest 힌트 + `state/touched_src_tests.flag` | 스크립트 **없음** |

로드 확인: Cursor **Hooks** 출력 채널. 로컬 검증은 프로젝트 루트 + Git Bash 권장.
